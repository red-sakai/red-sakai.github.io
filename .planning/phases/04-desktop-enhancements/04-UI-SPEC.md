## Interaction Contracts

### 1. Context Menu (Restoration) — D-16, D-17

**Behavior:**
- Right-click on desktop background → show context menu at cursor position
- Full item tree: View (Large Icons, Small Icons, List, Details), Sort By (Name, Size, Type, Date Modified), separator, Refresh, Paste, separator, New (Folder, Text Document), Properties, Display Settings
- **Hover-to-open submenus (D-17):**
  - Mouse enters parent item → 150ms delay → open submenu
  - Mouse moves to child submenu → keep submenu open (no close)
  - Mouse leaves parent OR child submenu → close submenu
  - Mouse enters a DIFFERENT parent → close current, open new
- Click on item with no children → call `item.action()` → close context menu
- Click on disabled item → no-op
- Click outside context menu → close
- Right-click on desktop while context menu is open → close current, open new at new position
- `Refresh` action increments `refreshTick` (triggers DesktopIcon blink animation)
- `View` actions change icon layout (Large Icons default → Small Icons/List/Details) — currently cosmetic
- `Sort By` actions reorder `DESKTOP_ICONS` array — updates icon rendering order
- `New > Folder` opens notepad / mock dialog (minimal — shows "New Folder" created)
- `New > Text Document` opens notepad / mock dialog
- `Properties` shows mock properties dialog
- `Display Settings` opens Control Panel program

**States:**
- **Default:** Hidden (not rendered)
- **Open:** Positioned at cursor, all items visible
- **Hovered (parent):** Parent highlighted, submenu appears after 150ms
- **Hovered (child):** Child item highlighted
- **Disabled:** Item text gray (`#808080`), no cursor pointer, no hover highlight

**Implementation:**
- Rewrite `handleItemClick` to NOT return early when `item.children` is truthy
- Add `hoveredParent` state + 150ms `useRef` timer in `ContextMenu.tsx`
- Add submenu rendering via `position: absolute; left: 100%; top: -3px` using existing `.win98-context-submenu` class
- Add `onMouseEnter`/`onMouseLeave` handlers to both parent items and submenu containers
- Non-interactive submenu items (Folder, Text Document) return early without close

**Refs:**
- CONTEXT.md D-16, D-17
- CONTEXT.md Specifics: "Refresh works (increments refreshTick)", "New Folder opens a notepad", "Display Settings opens Control Panel", "Properties shows a mock properties dialog", "View changes are cosmetic", "Sort By affects icon ordering"
- RESEARCH.md Patterns 1, Code Examples "Context Menu: Hover-to-Open Submenu Pattern"
- Existing CSS: `.win98-context-menu`, `.win98-context-item`, `.win98-context-submenu`, `.win98-context-separator`, `.win98-context-icon`, `.win98-context-label`, `.win98-context-arrow`, `.win98-context-item.disabled` — all exist and are ready

### 2. Internet Explorer Bugfix — D-18

**Fixes:**
1. **Stale closure in `navigate`:** Replace `historyIndex` state reads in callbacks with `useRef`-based pattern. Add `historyIndexRef` synced on every state change. Remove `historyIndex` from `useCallback` deps.
2. **Smart blocked timer:** Timer callback checks `loadingRef.current` before setting `blocked=true`. If `handleIframeLoad` already fired, loading is false and timer is a no-op.
3. **History title update:** Use `historyIndexRef.current` in `handleIframeLoad` instead of closure-captured `historyIndex`.
4. **Timer consolidation:** `handleNav("back"/"forward")` duplicates timer logic — consolidate into a shared helper.

**States:**
- **Loading:** "Loading..." indicator visible, iframe shows nothing or previous page
- **Loaded (normal):** iframe content displayed, loading indicator hidden
- **Blocked timer fired but loading completed:** No visible change (blocked never shows)
- **Blocked timer fired and still loading:** "This page cannot be displayed" screen shown
- **Refresh:** `iframeRef.current?.contentWindow?.location.reload()` — currently doesn't reset timer. Fix: clear and restart the blocked timer

**Refs:**
- CONTEXT.md D-18
- RESEARCH.md Pattern 2, Code Examples "IE Fix: Ref-Based History Index"
- RESEARCH.md Pitfall 1

**D-21: Shut Down Confirmation**

- Pressing OK plays shutdown sound if sound system is active (defer — sound not explicitly requested for dialog)

**States:**
- **Hidden:** `showShutDownDialog = false` (default)
- **Visible:** Centered modal, backdrop dims desktop, no interaction with desktop while visible

### 5. Game Station — D-22, D-23

**Behavior:**
- New program component: `GameStation.tsx` in `app/desktop/components/programs/`
- 6th desktop icon in `DESKTOP_ICONS` (id: `"games"`)
- Window displays game list as buttons
- Game data: hardcoded `DOS_GAMES` array with `{ id, title, archiveIdentifier, year }`
- First game: `{ id: "resident-evil", title: "Resident Evil (1996)", archiveIdentifier: "resident-evil_202406", year: "1996" }`
- Clicking a game → changes view to show archive.org embed iframe + back button header
- Back button → returns to game list view
- iframe sandbox: `allow-scripts allow-forms allow-same-origin allow-popups`
- iframe supports fullscreen (`allowFullScreen`)

**States:**
- **List view:** Title "Game Station", subtitle, game buttons
- **Game view:** Back button, game title bar, archive.org iframe filling remaining space
- **Loading (game):** iframe loading handled by archive.org (no custom loading state needed initially)
- **Blocked (game):** May reuse IE blocked pattern if archive.org embed fails (archive.org allows embedding by design — low risk per RESEARCH.md Environmental Considerations)

**Implementation notes:**
- Two-state rendering: `selectedGame ? gameView : listView`
- Expand type union in `useWindowManager.ts` — add `"games"` to `WindowState["component"]`
- Add title/icon mapping for `"games"` in `openWindow`
- Add `case "games"` to `renderProgram` in `DesktopShell.tsx`
- Add `{ id: "games", icon: "🎮", label: "Game Station" }` to `programList`
- Add `{ id: "games", icon: "🎮", label: "Game Station" }` to `DESKTOP_ICONS`
- Remove `{ id: "about", ... }` from `programList` (D-20)

**Refs:**

## Error & Edge Cases

| Case | Behavior |
|------|----------|
| IE navigate to invalid URL | `sanitizeUrl` returns null → no-op (existing behavior preserved) |
| IE navigate to blocked site | 10s timer fires only if loading still active → blocked screen with URL + frame explanation |
| IE cross-origin title read | `try/catch` in `handleIframeLoad` (cross-origin error is silent) |
| Game Station archive.org 404 | EM-DOSBOX doesn't load. Reuse IE blocked pattern or show error state. |
| Game Station archive.org identifier changes | Hardcoded array — update `archiveIdentifier` value at top of file |
| Context menu at screen edge | Submenu may overflow. Position at fixed screen coordinates with viewport clamping (current code positions at cursor — may need bounding) |
| Window manager union type missing `"games"` | TypeScript errors on `renderProgram` switch (no `default` case) |
| Shut Down dialog open while context menu open | Context menu closes on any click — dialog click closes menu first, then dialog shows |
