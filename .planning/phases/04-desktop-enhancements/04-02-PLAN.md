    1. **Add imports**: Add `useState` to the React import. Result: 
       `import { useEffect, useRef, useCallback, useState } from "react";`

    2. **No interface/props changes**: The `ContextMenuItem` and `ContextMenuProps` interfaces are correct as-is. Do not modify them.

    3. **Add state and refs** inside the component body, after `const ref = useRef<HTMLDivElement>(null);`:
       - `const [hoveredParent, setHoveredParent] = useState<number | null>(null);`
       - `const hoverTimerRef = useRef<ReturnType<typeof setTimeout> | null>(null);`

    4. **Add hover handler callbacks** after `const close`:
       
       `handleItemMouseEnter` — Accepts `(index: number, hasChildren: boolean)`:
       - If `!hasChildren`, return early
       - Clear existing timeout if `hoverTimerRef.current` is set
       - Set `hoverTimerRef.current = setTimeout(() => { setHoveredParent(index); }, 150)`

       `handleItemMouseLeave` — Accepts `(index: number)`:
       - Clear timeout if `hoverTimerRef.current` is set
       - Set `hoverTimerRef.current = null`
       - Close submenu if currently hovering this index: `setHoveredParent((current) => current === index ? null : current)`

    5. **Replace `handleItemClick`** (currently returns early if `item.children` is truthy):
       - Remove the `item.children` early return: only check `if (item.disabled) return;`
       - Call `item.action?.();` 
       - Call `close();`
       - Note: Items WITH children (like "View", "Sort By") will NOT fire any action on click — they only open submenus via hover. This is correct Win98 behavior. The click handler should still work for leaf items. For parent items, clicking does nothing (submenu is already open from hover). This matches Win98.

    6. **Replace the render loop** (lines 49-65). The new render loop must:

       a. Keep separator rendering the same (`if (item.separator) return <div className="win98-context-separator" />`)
       
       b. For non-separator items, render the item div:
          - Class: `"win98-context-item" + (item.disabled ? " disabled" : "")`
          - `onMouseEnter`: call `handleItemMouseEnter(i, !!item.children)`
          - `onMouseLeave`: call `handleItemMouseLeave(i)`
          - `onClick`: call `handleItemClick(item)` (preserve existing behavior — no longer early-returns for children)
          - Style: add `position: "relative"` to the item div (for submenu positioning)
       
       c. Inside the item div, render:
          - Icon span (`className="win98-context-icon"`) if `item.icon` exists
          - Label span (`className="win98-context-label"`) if `item.label` exists
          - Arrow span (`className="win98-context-arrow"`) if `item.children` exists — content: `"▶"` or `"▸"`
       
       d. **After** the item div closes (but still inside the `items.map` callback), conditionally render the submenu:
          ```tsx
          {item.children && hoveredParent === i && (
            <div
              className="win98-context-submenu"
              onMouseEnter={() => {
                /* Keep submenu open when mouse enters it — clear any pending close */
                if (hoverTimerRef.current) clearTimeout(hoverTimerRef.current);
              }}
              onMouseLeave={() => {
                /* Close submenu when mouse fully leaves it */
                setHoveredParent(null);
              }}
            >
              {item.children.map((child, ci) => {
                if (child.separator) return <div key={ci} className="win98-context-separator" />;
                return (
                  <div
                    key={ci}
                    className={`win98-context-item${child.disabled ? " disabled" : ""}`}
                    onClick={() => handleItemClick(child)}
                  >
                    {child.icon && <span className="win98-context-icon">{child.icon}</span>}
                    {child.label && <span className="win98-context-label">{child.label}</span>}
                  </div>
                );
              })}
            </div>
          )}
          ```

    7. **Preserve** the outside-click detection (`useEffect` with `mousedown` listener) exactly as-is.
    8. **Preserve** the root container's className, style, and `onMouseDown`.

    The key behavioral contract (from D-17 and UI-SPEC Interaction Contract §1):
    - Mouse entering a parent item → 150ms delay → submenu opens
    - Mouse moving to child submenu → keep submenu open
    - Mouse leaves parent OR child → close submenu
    - Mouse enters a DIFFERENT parent → close current, open new (naturally handled by `hoveredParent` state update)
    - Click on disabled item → no-op (handled by `handleItemClick` returning early)
    - Click on item with no children → call `action()` + close
