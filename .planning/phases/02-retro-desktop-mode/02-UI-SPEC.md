# Phase 02 — Retro Desktop Mode: UI Design Contract

> Visual and interaction contract for the Win98-style retro desktop enhancements.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> This phase uses a **custom retro design system** (not shadcn). All desktop components use `win98-*` CSS classes and inline styles.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — fully custom Win98 retro system |
| Preset | not applicable |
| Component library | none (custom-built: Window, Taskbar, StartMenu, DesktopIcon, ContextMenu, programs) |
| Icon library | Emoji only (📄 📁 🎨 ⚙️ 🌐 🏠 💻) |

## Spacing Scale

Win98 uses compact pixel-level spacing — NOT the standard 8-point web scale. All values are tightly packed to match the retro aesthetic.

| Token | Value | Usage |
|-------|-------|-------|
| xxs | 2px | Titlebar padding, border widths, button gaps |
| xs | 4px | Icon gaps, context menu item padding, start item padding |
| sm | 6px | Start item icon-label gap, context menu gap |
| md | 8px | Section padding inside group boxes (Control Panel) |
| lg | 12px | Control Panel content padding, group box content |
| xl | 16px | Window content white area default padding |
| 2xl | 20px | Desktop icon spacing between items |
| 3xl | 30px | Taskbar height (fixed), Start menu sidebar width |

**Exceptions:**
- **44px minimum touch targets** on mobile for icon buttons (handled via responsive CSS media query)
- **2px** borders used for 3D raised/sunken effects (white top/left, dark bottom/right)
- **3px** used in context menu item padding (`padding: 3px 20px 3px 4px`)
- **Resize handle** is 14×14px (not a multiple of 4)
- **Titlebar** height is 18px (not a multiple of 4)
- **Title buttons** are 16×14px (not multiples of 4)
- **Start button** height is 22px

## Interaction Contracts

### Desktop Icons (D-15, D-14)

| Behavior | Specification |
|----------|---------------|
| Icons shown | 4: My Portfolio (`🏠`), My Computer (`💻`), Paint (`🎨`), Control Panel (`⚙️`) |
| Icons removed | About Me (`📄`), Projects (`📁`) — deleted from `desktopIcons` array |
| Icon visibility | Filtered by `desktop-icons-vis` localStorage array — icons with unchecked boxes are hidden |
| Selection | Single click selects (blue highlight overlay + inverted emoji filter) |
| Open | Double-click opens corresponding program window (400ms threshold) |
| Mouse speed | Double-click threshold adjustable via Control Panel (200/400/600/800ms) |
| Refresh animation | Icons blink twice (`desktop-icon-blink` 150ms × 4 iterations) on context menu Refresh |
| Layout | CSS flex-wrap or grid container (NOT absolute positioning per icon) — icons reflow when hidden/shown |

### Context Menu (D-09)

| Behavior | Specification |
|----------|---------------|
| Trigger | Right-click on desktop background |
| Items displayed | Single item: **Refresh** (icon: `⟳`) |
| Items removed | View, Sort By, Paste, Properties, all separators |
| Refresh action | Increment `refreshTick` counter → triggers icon blink animation |
| Close | Click outside menu OR click any item |
| Position | Fixed at mouse click coordinates, with screen-edge boundary checking |

### Wallpaper System (D-01, D-02, D-03, D-04)

| Behavior | Specification |
|----------|---------------|
| Default | Solid `#008080` (Classic Teal) when no wallpaper configured |
| Type: Color | Inline `background` CSS property with hex value |
| Type: Preset | `background-image: url(/wallpapers/{filename})` — images from `public/wallpapers/` |
| Type: Imported | `background-image: url(data:...)` — base64 data URI from localStorage |
| Fit: Tile | `background-repeat: repeat; background-size: auto; background-position: 0 0` |
| Fit: Center | `background-repeat: no-repeat; background-size: auto; background-position: center center` |
| Fit: Stretch | `background-repeat: no-repeat; background-size: 100% 100%; background-position: 0 0` |
| Import limit | localStorage try-catch for `QuotaExceededError` (~5MB total) |
| Import feedback | Success: wallpaper appears. Failure: show "Storage full — remove some wallpapers" |
| Persistence | `desktop-wallpaper` key: `JSON.stringify({type, value, fit})` |

### Color Scheme Application (D-11)

| Behavior | Specification |
|----------|---------------|
| Mechanism | CSS Custom Properties set via `element.style.setProperty()` on desktop container |
| Properties set | `--titlebar-start`, `--titlebar-end`, `--taskbar-bg` |
| CSS `var()` fallback | `.win98-titlebar` uses `var(--titlebar-start, #000080)` and `var(--titlebar-end, #1084d0)` |
| Taskbar fallback | `.win98-taskbar` uses `var(--taskbar-bg, #c0c0c0)` |
| Persistence | `desktop-colorscheme` key: string ID (`"standard"`, `"rose"`, etc.) |

### Internet Explorer Browser (D-05, D-06, D-07, D-08)

| Behavior | Specification |
|----------|---------------|
| Window chrome | IE5/6-style: gray background, blue titlebar, raised 3D navigation buttons |
| Address bar | Text input with "Address:" label, font 11px MS Sans Serif |
| Navigation buttons | Back (←), Forward (→), Refresh (⟳), Stop (✕), Home (🏠) — rendered as raised 3D buttons |
| URL normalization | Auto-prepend `https://` if no protocol specified (D-08) |
| Protocol rejection | Block `javascript:`, `data:`, `file:`, `vbscript:` protocols (security) |
| Default page | `about:blank` (D-07) |
| History | Back/forward stack of URLs with index tracking |
| Loading indicator | Address bar or status area shows loading state |
| X-Frame-Options handling | 10-second timer + onLoad event — if neither fires, show blocked state |
| Blocked state UI | Win98-styled "This page cannot be displayed" error with explanation |
| iframe sandbox | `sandbox="allow-scripts allow-forms allow-same-origin"` |
| Component type | `"browser"` added to `WindowState.component` union type |

### Screen Resolution (D-12)

| Behavior | Specification |
|----------|---------------|
| Options | 640×480, 800×600, 1024×768 — radio buttons or list |
| Effect | Purely cosmetic — does NOT change actual rendering |
| Note | "Changes take effect after restart." shown below options |
| Persistence | `desktop-resolution` key: string ID |

### Mouse Settings (D-13)

| Behavior | Specification |
|----------|---------------|
| Swap buttons | Checkbox — when active, left/right click behavior swapped on DesktopIcon |
| Double-click speed | Slider: 200ms (fast), 400ms (default), 600ms (slow), 800ms (very slow) |
| Persistence | `desktop-swapbuttons`: `"true"` / `"false"`; `desktop-mousespeed`: millisecond string |

### Icon Visibility (D-14)

| Behavior | Specification |
|----------|---------------|
| Checkboxes | One per icon: My Portfolio, My Computer, Paint, Control Panel |
| Default | All checked (visible) |
| Filtering | DesktopShell filters `desktopIcons` array by `desktop-icons-vis` localStorage array |
| Persistence | `desktop-icons-vis`: `JSON.stringify(string[])` — array of visible icon IDs |

### Window Management

| Behavior | Specification |
|----------|---------------|
| Open | `openWindow(component)` creates new `WindowState` with incremented z-index |
| Focus | Click on window brings to front (max z-index) |
| Drag | Titlebar mousedown → track mouse → update x/y — releases on mouseup |
| Resize | Resize handle mousedown → track mouse → update width/height (min 200×100) |
| Minimize | Toggle `minimized` — window hidden but stays in `windows` array, shown in taskbar |
| Maximize | Toggle `maximized` — `position: fixed; inset: 0; height: calc(100% - 30px)` |
| Close | Set `animatingOut` → 150ms fade → remove from `windows` array |
| Cascade offset | Each program type gets a cascade offset: explorer=0, paint=15, controlpanel=30, browser=45, portfolio=60 |
