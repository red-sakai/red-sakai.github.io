<task type="auto">
  <name>Task 2: Simplify ContextMenu to flat item rendering</name>
  <files>app/desktop/components/ContextMenu.tsx</files>
  <action>
    Per D-09 (context menu shows ONLY "Refresh"), the ContextMenu component no longer needs submenu support. The DesktopShell will pass a single-item array: `[{ label: "Refresh", icon: "⟳", action: handleRefresh }]`.

    **What to change:**

    1. **Remove unused imports/state** (lines 3, 23-24):
       - Remove `useState` from the React import (keep `useEffect`, `useRef`, `useCallback`)
       - Remove `subLabel` state variable (`const [subLabel, setSubLabel] = useState<string | null>(null);`)
       - Remove `timerRef` ref (`const timerRef = useRef<ReturnType<typeof setTimeout> | undefined>(undefined);`)
       - Remove `showSub` function (lines 44-51)
       - Remove `renderSubmenu` function (lines 53-73)

    2. **Simplify the render method** (replace lines 82-98):
       Remove `onMouseEnter`, `onMouseLeave` handlers, submenu rendering, and arrow indicator from the item render loop.

       Simplified item render:
       ```tsx
       {items.map((item, i) => {
         if (item.separator) return <div key={i} className="win98-context-separator" />;
         return (
           <div
             key={i}
             className={`win98-context-item${item.disabled ? " disabled" : ""}`}
             onClick={() => {
               if (item.disabled) return;
               item.action?.();
               close();
             }}
           >
             {item.icon && <span className="win98-context-icon">{item.icon}</span>}
             {item.label && <span className="win98-context-label">{item.label}</span>}
           </div>
         );
       })}
       ```

       Note: `item.children` items are simply rendered as regular items (the `children` field remains in the interface for type compatibility but is not used in rendering logic). This ensures backward compatibility with the ContextMenuItem interface that DesktopShell uses.

    3. **Keep unchanged:**
       - `ContextMenuItem` interface (keep children field — type compat for DesktopShell that may pass items with it)
       - `ContextMenuProps` interface
       - `close` callback
       - Click-outside handler (`useEffect` with `mousedown` event listener)
       - `handleItemClick` function (keep as-is — it already handles disabled/children correctly)
       - Container element structure (`className="win98-context-menu"`, fixed positioning, z-index)

    **What to avoid:** Do not change the CSS classes or styling. The `win98-context-arrow` CSS class remains in desktop.css but is no longer used in JSX — it can be cleaned up later but is harmless when unused. Keep the `children` field on `ContextMenuItem` interface to avoid breaking type checks in DesktopShell.
  </action>
