    **URL sanitization function** (inside component, before handlers):
    ```typescript
    function sanitizeUrl(input: string): string | null {
      const trimmed = input.trim();
      if (!trimmed || trimmed === "about:blank") return "about:blank";
      if (/^(javascript|data|file|vbscript):/i.test(trimmed)) return null; // D-08 security
      const url = /^https?:\/\//i.test(trimmed) ? trimmed : `https://${trimmed}`; // D-08 auto-prepend
      try { new URL(url); return url; }
      catch { return null; }
    }
    ```
    Per D-08: auto-prepend `https://` if no protocol specified.
    Per security requirements: reject javascript:/data:/file:/vbscript: protocols.
    Per D-07: default page is about:blank.

    **navigate function** (useCallback):
    - Calls sanitizeUrl on the input
    - If sanitizeUrl returns null, do nothing (invalid URL)
    - Updates address state with the sanitized URL (empty string for about:blank)
    - Sets currentUrl to the sanitized URL
    - Resets blocked to false, sets loading to true
    - Trims future history if navigated from a mid-point in history (slice to historyIndex + 1)
    - Pushes new entry, increments historyIndex
    - Starts a 10-second timer: if iframe's onLoad hasn't fired by then, set blocked=true

    **Navigation handlers** (useCallback):
    - `handleNav("back")`: If historyIndex > 0, decrement index, set currentUrl/address from history entry, reset blocked
    - `handleNav("forward")`: If historyIndex < history.length - 1, increment index, set currentUrl/address, reset blocked
    - `handleNav("refresh")`: Reload iframe via `iframeRef.current?.contentWindow?.location.reload()`
    - `handleNav("stop")`: Call `iframeRef.current?.contentWindow?.stop()`, set loading to false, clear timer
    - `handleNav("home")`: Navigate to "about:blank"
    - Back/forward buttons disabled at bounds (historyIndex <= 0 for back, historyIndex >= history.length - 1 for forward)

    **Iframe event handlers:**
    - `handleIframeLoad`: Set loading false, clear blocked timer, clear blocked state. Try to read iframe title (cross-origin will throw — catch silently).
    - `handleIframeError`: Set blocked = true, loading = false, clear timer.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| URL input → iframe src | User-provided URL string crosses into iframe navigation — primary injection vector |
| iframe external content | Third-party page content rendered inside portfolio site |
| localStorage | QuotaExceededError could break all settings if unhandled |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04 | Tampering | sanitizeUrl() — protocol injection | mitigate | Reject javascript:/data:/file:/vbscript: protocols before setting iframe src. Use `new URL()` constructor for validation. Per RESEARCH.md security domain and ASVS V5 Input Validation. |
| T-02-05 | Tampering | iframe clickjacking / sandbox escape | mitigate | iframe uses `sandbox="allow-scripts allow-forms allow-same-origin"` — prevents popups, plugin execution, and top-level navigation. Per RESEARCH.md security domain. |
| T-02-06 | Information Disclosure | Mixed content (HTTPS site loading HTTP iframe) | mitigate | Auto-prepend `https://` per D-08. Browser will block HTTP iframe on HTTPS origin — this is expected and handled by the blocked fallback UI. |
| T-02-07 | Denial of Service | Timer-based blocked detection | accept | 10-second timer is cleared on successful load. If timer fires, sets blocked state. No resource exhaustion risk. |
| T-02-SC | Tampering | npm installs | accept | No new packages in this phase — all browser APIs |
</threat_model>
