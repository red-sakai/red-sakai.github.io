## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| localStorage reads → component state | Wallpaper config, color scheme, icon visibility parsed from JSON |
| wallpaper value → CSS background-image | Can be color hex, URL path, or data URI — all safe for CSS background-image |
| CSS variable injection | Runtime override of CSS custom properties via style.setProperty() |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-11 | Tampering | localStorage JSON.parse | mitigate | All localStorage reads wrapped in try-catch with safe defaults per D-03 (teal wallpaper, standard scheme, all icons visible, 400ms speed) |
| T-02-12 | Tampering | CSS variable injection | accept | Visual rendering only — no security impact. Malicious values would look wrong but cannot execute code. |
| T-02-13 | Tampering | DesktopIcon click handler — swapButtons | accept | Boolean prop controls click behavior. Max impact: user confusion if accidentally toggled. |
| T-02-SC | Tampering | npm installs | accept | No new packages in this phase |
