    **5. LOCAL STORAGE PERSISTENCE ON IMPORT:**

    On mount, scan localStorage for `imported-wallpaper-*` keys and populate `localImported`:
    ```typescript
    useEffect(() => {
      const imported: {id:string, label:string, dataUri:string}[] = [];
      for (let i = 0; i < localStorage.length; i++) {
        const key = localStorage.key(i);
        if (key?.startsWith("imported-wallpaper-")) {
          const val = localStorage.getItem(key);
          if (val) imported.push({ id: key, label: key.replace("imported-wallpaper-", "Imported "), dataUri: val });
        }
      }
      setLocalImported(imported);
    }, []);
    ```

    Import handler (D-04, D-10):
    ```typescript
    const handleImport = (e: React.ChangeEvent<HTMLInputElement>) => {
      const file = e.target.files?.[0];
      if (!file) return;
      const reader = new FileReader();
      reader.onloadend = () => {
        const dataUri = reader.result as string;
        try {
          const key = `imported-wallpaper-${Date.now()}`;
          localStorage.setItem(key, dataUri);
          // Update local state to show the new import
          setLocalImported(prev => [...prev, { id: key, label: `Imported ${file.name}`, dataUri }]);
          // Auto-apply as wallpaper
          onWallpaperChange({ type: "imported", value: dataUri, fit: wallpaper.fit });
        } catch (err) {
          if (err instanceof DOMException && err.name === "QuotaExceededError") {
            alert("Storage full — remove some wallpapers first.");
          }
        }
      };
      reader.readAsDataURL(file);
    };
    ```

| Boundary | Description |
|----------|-------------|
| File picker → FileReader → localStorage | User-uploaded image files read as base64 data URIs and stored in localStorage |
| localStorage reads → component state | Stored wallpaper config parsed from JSON — malformed data could crash JSON.parse |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-08 | Tampering | FileReader — uploaded file handling | mitigate | FileReader.readAsDataURL() produces safe base64 data URIs. Images are only used as CSS backgrounds — no script execution possible from data: URIs in background-image. |
| T-02-09 | Tampering | localStorage JSON.parse | mitigate | All localStorage reads wrapped in try-catch with default fallback per D-03/D-04. Malformed JSON cannot crash the component. |
| T-02-10 | Denial of Service | localStorage QuotaExceededError | mitigate | Every localStorage.setItem() wrapped in try-catch. On quota error, user sees "Storage full" alert. Other settings still persist. |
| T-02-SC | Tampering | npm installs | accept | No new packages in this phase |
